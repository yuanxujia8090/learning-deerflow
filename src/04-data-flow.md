# 数据与请求流程

## 完整请求生命周期

```
用户（浏览器 / IM / CLI）
    │
    ▼
┌──────────────────────────────────────────────────────┐
│                   1. 请求入口                          │
│                                                      │
│   ┌─ 浏览器 ─┐   ┌─ IM 渠道 ─┐   ┌─ Python SDK ─┐  │
│   │  :3000   │   │ Telegram  │   │ client.py   │  │
│   │  :2026   │   │ Slack/... │   │             │  │
│   └──────────┘   └───────────┘   └─────────────┘  │
│                                                     │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────────┐
│                  2. 网关层                             │
│                                                      │
│   AuthMiddleware → CSRFMiddleware → CORSMiddleware   │
│                           ↓                          │
│                   路由分发                              │
│   /api/runs/*    → runs.py     （无状态运行）          │
│   /api/threads/  → thread_runs.py（有状态运行）       │
│   /api/models    → models.py                          │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────────┐
│                 3. 运行时层                             │
│                                                      │
│   run_agent()                                        │
│       ↓                                              │
│   创建 RunRecord（运行记录）                            │
│       ↓                                              │
│   构建 RunContext（checkpointer, store...）           │
│       ↓                                              │
│   调用 agent_factory(config) → 构建代理图             │
│       ↓                                              │
│   配置检查点 + 中断节点                                │
│       ↓                                              │
│   agent.astream() 开始执行                            │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────────┐
│                 4. 代理执行层                           │
│                                                      │
│   ① 中间件管道（按序执行）                               │
│   ├── before_agent 阶段                                │
│   │   ├── ThreadDataMiddleware   → 初始化目录          │
│   │   ├── UploadsMiddleware      → 处理上传            │
│   │   ├── SandboxMiddleware      → 获取沙箱            │
│   │   ├── SummarizationMiddleware → 上下文截断         │
│   │   ├── MemoryMiddleware       → 注入记忆            │
│   │   └── ...                                          │
│   │                                                    │
│   ② LLM 推理                                          │
│   ├── 系统提示词 + 用户消息 + 上下文                    │
│   ├── LLM 选择工具或生成响应                            │
│   └── 返回 AI 消息或工具调用                            │
│                                                       │
│   ③ 工具执行（循环直到 LLM 返回最终响应）                │
│   ├── 内置工具（present_file, ask_clarification...）    │
│   ├── MCP 工具（通过 MCP 客户端）                       │
│   ├── 沙箱工具（bash, read, write...）                  │
│   └── ACP 工具（多代理工具）                            │
│                                                       │
│   ④ after_agent 阶段                                  │
│   ├── TitleMiddleware       → 生成标题                 │
│   ├── MemoryMiddleware      → 排队记忆更新             │
│   ├── ClarificationMiddleware → 处理澄清              │
│   └── ...                                             │
│                                                       │
│   ⑤ 子代理委派（如果需要）                               │
│   ├── SubagentExecutor 创建子代理                       │
│   ├── 独立上下文 + 独立模型                              │
│   ├── 并行或串行执行                                    │
│   └── 结果返回主导代理                                  │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────────┐
│                  5. 流式输出层                          │
│                                                      │
│   StreamBridge → SSE 事件流                           │
│       ↓                                              │
│   event: metadata    (run_id, thread_id)             │
│   event: values      (状态快照)                        │
│   event: messages    (AI 消息增量)                     │
│   event: end         (运行结束)                        │
│       ↓                                              │
│   Nginx 转发 → 浏览器 EventSource                    │
│   或 → IM 渠道消息发送                                 │
└──────────────────────────────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────────┐
│                  6. 持久化层                            │
│                                                      │
│   检查点 → 保存线程状态（可恢复）                        │
│   记忆  → 异步更新记忆文件                              │
│   日志  → 记录运行日志和 Token 使用                     │
│   产物  → 文件保存到 outputs 目录                       │
└──────────────────────────────────────────────────────┘
```

## 典型用户请求的数据流

### 简单对话（无工具调用）

```
用户: "你好"
    ↓
core/api/  →  POST /api/runs
    ↓
Gateway → run_agent → 中间件 → LLM
    ↓
LLM 直接响应（无需工具）
    ↓
SSE: messages + end
    ↓
前端渲染: "你好！有什么可以帮助你的？"
```

### 带工具调用的对话

```
用户: "搜索 AI Agent 的最新发展"
    ↓
Gateway → 中间件链 → LLM
    ↓
LLM 决定调用 tavily_search（工具调用）
    ↓
工具执行 → 结果返回 LLM
    ↓
LLM 继续推理 → 是否需要更多工具？
    ↓
如果需要 → 再次工具调用
如果不需要 → 生成最终响应
    ↓
SSE: 流式返回回答
    ↓
同时：记忆中间件排队 → 异步更新记忆
```

### 复杂任务（子代理委派）

```
用户: "写一份 AI Agent 行业报告"
    ↓
主导代理分析任务 → 决定委派
    ↓
SubagentExecutor 创建 3 个子代理：
├── 子代理A：市场调研任务（并行）
├── 子代理B：技术分析任务（并行）
└── 子代理C：竞争分析任务（并行）
    ↓
每个子代理独立执行：
├── 独立的上下文
├── 独立的工具集
└── 独立的结果
    ↓
子代理结果返回主导代理
    ↓
主导代理综合三份报告
    ↓
生成最终报告并写入文件
    ↓
SSE: 流式返回 + 文件链接
```

## 嵌入模式数据流（Python Client）

```python
from deerflow.client import DeerFlowClient

client = DeerFlowClient()

# 直接用 Python 调用，不走 HTTP
response = client.chat("分析这篇论文", thread_id="my-thread")

# 内部流程（与 HTTP 请求相同，只是省去网络层）
client.chat()
    ↓
run_agent()  # 直接调用运行时
    ↓
[同上 2-6 步]
```
