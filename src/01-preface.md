# 前言

## 关于 DeerFlow

DeerFlow（Deep Exploration and Efficient Research Flow）是字节跳动开源的一款**超级代理运行时**（Super Agent Harness）。它不是传统意义上的 Web 框架或库，而是一个能让 AI 代理真正"干活"的基础设施——提供文件系统、持久记忆、沙箱执行、技能扩展、子代理委派等能力。

### 本项目面向谁？

如果你：
- 有**前端开发经验**，想进入 AI Agent 领域
- 需要在 DeerFlow 基础上做**二次开发与封装**
- 想理解 AI 代理的**完整数据流和核心架构**
- 有**企业级部署**需求

那么这份指南就是为你准备的。

### 学习目标

1. **熟悉掌握 DeerFlow**，具备二次开发和封装能力
2. **理解完整数据流程、核心模块、能力边界**
3. **掌握企业级部署与维护**相关知识

### 知识预备

你不需要 Python 或后端经验，但以下前端知识会让你更容易上手：

- **React 组件化思维** → 理解中间件管道
- **前端状态管理**（Redux/Zustand） → 理解 ThreadState
- **REST API / GraphQL** 使用经验 → 理解 Gateway API
- **Docker 基础** → 理解部署

## 复习习题

1. （初级）DeerFlow 的全称是什么？它由哪个公司开源？

2. （中级）本书将前端的状态管理库（Redux/Zustand）类比为 DeerFlow 的哪个概念？这对理解 Agent 系统有什么帮助？

3. （中级）DeerFlow 有四个核心理念：Skill、Sub-Agent、Sandbox 和 Memory，试用自己的话解释每个概念的作用。

4. （高级）参考材料提到 DeerFlow 采用"Harness"定位，这意味着它更适合作为基础设施层。你能举出一个具体的二次开发场景，说明这种设计带来的优势？

5. （中级）DeerFlow 2.0 引入了 Progressive Skill Loading（渐进式技能加载），为什么需要在 Agent 运行时中实现按需加载而非一次性全部加载？
